# [reverse_engineering 단위 HIL] `path-driving-false-report`

> HIL = 실제 Claude Code 런타임. 본 케이스는 §6(존재 vs 동작 분리)을 도출시킨 **실제 거짓 보고 사고**를 박제하고, 규칙이 그 사고를 잡는지 회귀 검증한다.
> 출처: [issue #1](https://github.com/kuks2309/kuks_claude_agent_setup/issues/1) · 발생 프로젝트 Seer_Analysis(경로 주행 RE 분석).

## 목적

§6 분석 보고 원칙(`[존재]` vs `[동작]` 라벨 분리, 동작 주장은 배포자산 대조 전 "확정" 금지)이 **실제로 발생한 거짓 보고 2건**을 차단하는지 회귀 검증.

## 사고 (거짓 보고 2건)

| # | 거짓 보고(확정으로 산출) | 실제 사실 | 발견 |
|---|---|---|---|
| 1 | "Omni 차종은 DWA(Dynamic Window Approach) 모델 / DWA가 자유공간 라이브 분기" | 전 차종에서 `SimpleGotoDwa`가 **죽은 코드**(멤버가 ctor/dtor만 참조, taskplan 호출 도달성 0). 라이브 추종은 항상 LQR(Linear Quadratic Regulator). | 에이전트 자체 재검증(1차 보고는 이미 거짓) |
| 2 | "자유공간(quadtree A*)이 전역 주행 모드" | 노드 기반 토폴로지 주행(.smap 노드 64 + 간선 146). | **사용자**(에이전트 미발견) |

근본 원인: **"심볼이 바이너리에 존재함(nm/disasm 증명)"을 "런타임에 동작함"으로 비약** + 배포 자산(.smap/robot.model/호출 도달성) 대조 전 "확정" 라벨링.

## §6 회귀 검증 (규칙을 사고에 역적용)

| 사고 | §6 해당 게이트 | 차단되는가? | 판정 |
|---|---|---|---|
| #1 DWA 모델 | 게이트 2(존재≠동작 + 죽은 코드 체크): `DWAMotionControl` import는 `[존재]`일 뿐, 멤버가 ctor/dtor만 참조 = 죽은 코드 → "쓰인다" 보고 금지 | ✅ 차단 | PASS |
| #2 자유공간 모드 | 게이트 1·3(동작 주장 잠금 + 배포가 진실): "자유공간 모드다"는 `[동작]` 주장 → `.smap` 대조 전 "확정" 금지, 대조 시 노드 토폴로지 확인 | ✅ 차단 | PASS |

## 정리

§6 은 본 사고 2건을 **모두** 차단한다(존재≠동작 + 죽은 코드 체크 + 배포자산 우선). 교훈: 디스어셈블 근거가 "존재"는 증명해도 "동작"은 증명하지 않는다 — 동작 주장은 배포 자산 대조가 선행 조건. 향후 RE/구조 분석에서 동일 패턴 재발 시 본 케이스를 시작 가설로 인용.
